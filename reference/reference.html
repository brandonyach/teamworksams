Function and Class Reference

Function and Class Reference#

This section provides detailed documentation for all teamworksams functions and classes, organized by module. Each entry links to a dedicated page with full parameter details, usage examples, return values, and error conditions. Use this reference to explore specific functionality, complemented by the vignettes section for practical workflows.

Credentials and Authentication#

  • get_client(): Create or retrieve an authenticated client for API interactions.

  • login(): Authenticate with an AMS instance and return session details.

  • AMSClient: Client class for handling AMS API requests and authentication.

  • AMSError: Custom exception for AMS API errors with descriptive messages.

  • LoginOption: Configuration options for the login process.

User Management#

  • get_user(): Retrieve user data as a pandas DataFrame, with optional filtering.

  • get_group(): Fetch group data as a pandas DataFrame.

  • edit_user(): Update user fields for existing accounts.

  • create_user(): Create new user accounts.

  • UserOption: Configuration options for user-related functions.

  • GroupOption: Configuration options for group-related functions.

  • UserFilter: Filter object for narrowing user data queries.

Data Export#

  • get_event_data(): Retrieve event data from an AMS Event Form within a date range.

  • sync_event_data(): Fetch event data modified since the last synchronization time.

  • get_profile_data(): Retrieve profile data from an AMS Profile Form.

  • EventFilter: Filter object for event data queries.

  • SyncEventFilter: Filter object for synchronized event queries.

  • ProfileFilter: Filter object for profile data queries.

  • EventOption: Configuration options for event data retrieval.

  • SyncEventOption: Configuration options for synchronized event retrieval.

  • ProfileOption: Configuration options for profile data retrieval.

Data Import#

  • insert_event_data(): Insert new event records into an AMS Event Form.

  • update_event_data(): Update existing event records in an AMS Event Form.

  • upsert_event_data(): Insert and update event records in an AMS Event Form.

  • upsert_profile_data(): Upsert profile data in an AMS Profile Form.

  • InsertEventOption: Configuration options for event insertion.

  • UpdateEventOption: Configuration options for event updates.

  • UpsertEventOption: Configuration options for event upserts.

  • UpsertProfileOption: Configuration options for profile upserts.

Database Operations#

  • get_database(): Retrieve database entries from an AMS database form.

  • delete_database_entry(): Delete a specific database entry by ID.

  • insert_database_entry(): Insert new database entries into a form.

  • update_database_entry(): Update existing database entries in a form.

  • GetDatabaseOption: Configuration options for database retrieval.

  • InsertDatabaseOption: Configuration options for database insertion.

  • UpdateDatabaseOption: Configuration options for database updates.

File Management#

  • upload_and_attach_to_events(): Upload files and attach to events in an AMS Event Form.

  • upload_and_attach_to_avatars(): Upload images and attach as user profile avatars.

  • FileUploadOption: Configuration options for file upload operations.

Form Management#

  • get_forms(): Fetch a list of forms accessible to the user.

  • get_form_schema(): Fetch and summarize a specific form’s schema.

  • FormOption: Configuration options for form metadata and schema retrieval.

Deleting Data#

  • delete_event_data(): Delete a single event from an AMS instance.

  • delete_multiple_events(): Delete multiple events from an AMS instance.

  • DeleteEventOption: Configuration options for event deletion.
